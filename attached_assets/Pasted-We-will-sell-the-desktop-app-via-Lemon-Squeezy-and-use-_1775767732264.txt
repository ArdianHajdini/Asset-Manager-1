We will sell the desktop app via Lemon Squeezy and use Lemon Squeezy license keys as the source of truth.

Please implement a simple licensing system for the Tauri desktop app.

GOAL
- Every buyer gets a Lemon Squeezy license key automatically after purchase
- The app asks for the key on first launch
- The app activates and validates the license using Lemon Squeezy License API
- The app stores the license locally
- The app supports a short offline grace period
- No custom random key generation

IMPLEMENTATION REQUIREMENTS

1. First-run license screen
- Show a clean activation screen on first launch
- Ask for:
  - license key
- optional: prefill from deep link or launch argument if available
- do not require the user to type the key again after successful activation

2. Lemon Squeezy activation flow
Use the official License API:
- POST /v1/licenses/activate
- send:
  - license_key
  - instance_name

Store locally after success:
- license_key
- instance_id
- last_validated_at
- instance_name

3. Validation flow
On later app starts:
- call POST /v1/licenses/validate
- send:
  - license_key
  - instance_id

If valid:
- allow app usage
- update last_validated_at

If invalid:
- block premium/full app usage
- show a clear German error

4. Offline grace period
- if online validation fails because of no internet
- allow app usage for 7 days after last successful validation
- after that require online validation again

5. Deactivation support
Add a settings button:
- "Lizenz auf diesem Gerät deaktivieren"

Use:
- POST /v1/licenses/deactivate
- send:
  - license_key
  - instance_id

Then clear local license data.

6. Security
- do not generate custom fake keys
- use Lemon Squeezy as source of truth
- keep implementation simple
- if possible, avoid exposing sensitive store API credentials in the frontend
- if needed, create a tiny validation proxy/backend later, but for now structure the code clearly

7. Code structure
Please create:
- src/services/licenseService.ts
- src/pages/LicensePage.tsx
- local license storage helper
- guard in app startup so the main app opens only when license is active or within grace period

8. German UX text examples
- "Lizenz aktivieren"
- "Lizenzschlüssel eingeben"
- "Lizenz erfolgreich aktiviert"
- "Lizenz konnte nicht validiert werden"
- "Offline-Modus: Lizenzprüfung zuletzt erfolgreich am ..."
- "Die Offline-Frist ist abgelaufen. Bitte verbinde dich mit dem Internet."

9. Documentation
Create:
- lemonsqueezy-license-setup.md

Explain:
- what I must configure in Lemon Squeezy
- where to enable license keys
- activation limit recommendation
- which environment variables are needed
- how activation/validation/deactivation works
