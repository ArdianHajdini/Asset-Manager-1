Erstelle eine benutzerfreundliche Windows-Desktop-App als lokalen CS2 Demo Viewer / Demo Manager. Die App soll komplett offline funktionieren und keine FACEIT-API oder Login benötigen. Sie soll einfach dazu dienen, bereits heruntergeladene CS2-Demo-Dateien auf dem PC bequem zu verwalten und mit einem Klick in Counter-Strike 2 zu öffnen.

Wichtige Klarstellung:
Die App soll kein eigenes 3D-Rendering der Demo bauen, sondern ein lokaler Launcher und Manager für CS2-Demos sein. Die eigentliche Wiedergabe erfolgt weiterhin im installierten Spiel Counter-Strike 2. Die App soll den komplizierten manuellen Ablauf für den Nutzer stark vereinfachen.

Technologie:
- Baue die App als Windows-Desktop-App
- Bevorzugt mit Tauri + React + TypeScript
- Modernes, sauberes, sehr einfaches UI
- Fokus auf Stabilität und Anfängerfreundlichkeit
- Der Code soll klar strukturiert, kommentiert und erweiterbar sein

Zielgruppe:
Normale CS2-Spieler, die Demos lokal öffnen wollen, ohne jedes Mal Dateinamen zu kopieren, Ordner zu durchsuchen oder manuell Konsolenbefehle einzugeben.

Hauptfunktionen:
1. Demo-Datei auswählen
   - Unterstütze .dem und .dem.gz Dateien
   - Unterstütze Drag & Drop
   - Unterstütze Dateiauswahl per Button

2. Automatisches Entpacken
   - Wenn eine .dem.gz Datei gewählt wird, soll die App sie automatisch entpacken
   - Die entpackte .dem Datei soll in einem lokalen Demo-Ordner gespeichert werden
   - Der Nutzer soll den Zielordner in den Einstellungen ändern können

3. Demo-Bibliothek
   - Zeige eine Liste aller verfügbaren lokalen Demos
   - Pro Eintrag: Dateiname, Dateigröße, Änderungsdatum, Speicherort
   - Suchfeld zum Filtern nach Dateinamen
   - Buttons pro Demo:
     - Öffnen in CS2
     - Ordner öffnen
     - Umbenennen
     - Löschen

4. CS2-Erkennung
   - Versuche automatisch den Installationspfad von Steam und Counter-Strike 2 unter Windows zu finden
   - Falls die automatische Erkennung fehlschlägt, soll der Nutzer den Pfad manuell setzen können
   - Zeige klar an, ob CS2 gefunden wurde oder nicht

5. Demo in CS2 starten
   - Implementiere eine zuverlässige Funktion „In CS2 öffnen“
   - Die App soll CS2 starten oder, falls sinnvoller, Steam mit passenden Startparametern öffnen
   - Ziel: Die ausgewählte Demo soll für den Nutzer möglichst mit einem Klick gestartet werden
   - Wenn eine vollständige Direktübergabe technisch nicht sauber möglich ist, implementiere einen robusten Fallback:
     - Kopiere automatisch den passenden playdemo-Befehl in die Zwischenablage
     - Zeige dem Nutzer danach klare Schritt-für-Schritt-Anweisungen an

6. Einstellungen
   - Demo-Ordner festlegen
   - CS2-/Steam-Pfad festlegen
   - Automatisch .dem.gz entpacken: an/aus
   - Nach Import automatisch zur Bibliothek hinzufügen
   - Alte Demos automatisch löschen: optional vorbereiten, aber einfach halten

7. Benutzeroberfläche
   - Sehr einfach
   - Startseite mit:
     - „Demo auswählen“
     - „Demo per Drag & Drop hier ablegen“
     - „Bibliothek öffnen“
   - Klare Statusanzeigen:
     - Datei importiert
     - Entpackt
     - CS2 gefunden
     - Demo gestartet
     - Fehler beim Start
   - Keine technische Überladung
   - Keine unnötigen Menüs

8. Fehlerbehandlung
   - Freundliche Fehlermeldungen auf Deutsch
   - Beispiele:
     - „CS2 wurde nicht gefunden.“
     - „Die Demo-Datei konnte nicht entpackt werden.“
     - „Diese Datei wird nicht unterstützt.“
     - „Die Demo konnte nicht automatisch gestartet werden. Der Befehl wurde in die Zwischenablage kopiert.“
   - Keine rohen technischen Fehler direkt dem Nutzer zeigen

9. Code-Struktur
   - Saubere Trennung zwischen UI, Dateiverwaltung, Entpacken, Pfaderkennung und Spielstart
   - Erstelle wiederverwendbare Services / Module
   - Verwende TypeScript-Typen sauber
   - Schreibe gut lesbaren Code mit kurzen Erklärkommentaren

10. Ausgabe
   - Liefere den vollständigen Projektcode
   - Zeige die Ordnerstruktur
   - Erkläre kurz, wie man das Projekt lokal startet
   - Erstelle außerdem eine einfache README auf Deutsch
   - Wenn nötig, simuliere anfangs den Start von CS2 mit einem Platzhaltermechanismus, aber strukturiere den Code so, dass die echte Startlogik leicht ergänzt werden kann

Wichtige UX-Regeln:
- Maximal einfache Bedienung
- So wenig Klicks wie möglich
- Der Nutzer soll nie Dateinamen manuell eintippen müssen
- Der Nutzer soll sofort verstehen, was zu tun ist
- Fokus auf Windows-PC und lokale Dateien
- Keine Online-Funktionen, kein Login, keine Cloud

Zusatz:
Bitte beginne mit einer MVP-Version, die zuverlässig folgende Kernpunkte kann:
- .dem und .dem.gz importieren
- .dem.gz entpacken
- Demos in einer Bibliothek anzeigen
- CS2-/Steam-Pfad speichern
- „In CS2 öffnen“-Button mit bestmöglicher Startlogik oder sauberem Fallback

Arbeite pragmatisch. Lieber eine einfache, stabile App als zu viele unfertige Features.
