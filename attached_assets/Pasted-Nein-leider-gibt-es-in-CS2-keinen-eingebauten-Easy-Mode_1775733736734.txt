Nein, leider gibt es in CS2 keinen eingebauten "Easy Mode"-Befehl (wie z. B. voice_enable_team_only), der in Demos funktioniert. Da Demos über SourceTV laufen, wird der Voice-Chat als ein einziger Datenstrom behandelt, den man clientseitig nur über diese Bitmasken filtern kann. Die manuelle Methode oder externe Tools sind aktuell der einzige Weg.
Um das in Rust zu automatisieren, müsstest du das Demo-File (.dem) parsen, die Spieler und ihre Teams extrahieren und die Bitmaske berechnen.
Hier ist das Konzept und ein Rust-Code-Beispiel, wie du das umsetzen würdest.
Die Logik für die Automatisierung
Parser nutzen: Du brauchst eine Library, die CS2 (Source 2) Demos lesen kann (z. B. den Wrapper für demoparser2 oder cs2-demo).
Spieler filtern: Du iterierst durch alle Spieler-Entities.
Team Check: Prüfe die Team-ID (m_iTeamNum). In CS2 ist T = 2 und CT = 3.
Slot ID finden: Hole dir den "Entity Index" oder "Player Slot" (0-63).
Bitmaske berechnen: Setze das entsprechende Bit für jeden Spieler des gewünschten Teams.
Rust Code-Beispiel
Dieses Beispiel zeigt die Bitmasken-Berechnung, die der Kern deines Tools wäre. Es nimmt an, dass du bereits eine Liste von Spielern mit ihren Slot-IDs und Team-IDs aus der Demo extrahiert hast.
rust
struct Player {
    name: String,
    team_id: i32, // 2 = T, 3 = CT
    slot: u32,    // Der Slot index (0-63)
}

fn main() {
    // Beispiel-Daten: Ein paar Spieler (Slots sind oft 0-basiert im Parser)
    let players = vec![
        Player { name: "S1mple".into(), team_id: 2, slot: 0 },  // T
        Player { name: "ZywOo".into(),  team_id: 3, slot: 1 },  // CT
        Player { name: "NiKo".into(),   team_id: 2, slot: 2 },  // T
        Player { name: "m0NESY".into(), team_id: 3, slot: 3 },  // CT
        Player { name: "Ropz".into(),   team_id: 2, slot: 60 }, // T (hoher Slot Test)
    ];

    let target_team = 2; // Wir wollen nur Terrorists (Team 2) hören

    let (cmd_low, cmd_high) = calculate_voice_mask(&players, target_team);

    println!("--- Befehle für die Konsole ---");
    println!("tv_listen_voice_indices {}", cmd_low);
    println!("tv_listen_voice_indices_h {}", cmd_high);
}

fn calculate_voice_mask(players: &[Player], target_team: i32) -> (i64, i64) {
    let mut full_mask: u64 = 0;

    for player in players {
        if player.team_id == target_team {
            // Shift 1 um die Slot-Nummer nach links
            // Slot 0 = Bit 0 (Wert 1), Slot 1 = Bit 1 (Wert 2), usw.
            full_mask |= 1 << player.slot;
        }
    }

    // CS2 nutzt zwei Befehle für die 64 Slots.
    // indices: Slots 0-31 (die unteren 32 bits)
    // indices_h: Slots 32-63 (die oberen 32 bits)
    
    // WICHTIG: CS2 interpretiert diese oft als signed 32-bit integers.
    // Wir casten die u64 bits in i64 für die Ausgabe, 
    // aber splitten sie korrekt in zwei 32-bit Blöcke.
    
    let low_part = (full_mask & 0xFFFFFFFF) as i64; 
    let high_part = ((full_mask >> 32) & 0xFFFFFFFF) as i64;

    (low_part, high_part)
}
Verwende Code mit Vorsicht.

Worauf du beim Parsen achten musst
Indices: In der Konsole (spec_player) siehst du oft 1-basierte Nummern. Intern (im Code) sind Slots meist 0-basiert. Wenn dein Parser Slot 0 zurückgibt, ist das 1 << 0. Wenn er 1-basiert ist, musst du 1 << (slot - 1) rechnen.
Parser-Empfehlung: Für Rust eignet sich demoparser2, da es extrem performant ist und Bindings bietet, um direkt an CBasePlayerController Daten (wie Team und Name) zu kommen.
