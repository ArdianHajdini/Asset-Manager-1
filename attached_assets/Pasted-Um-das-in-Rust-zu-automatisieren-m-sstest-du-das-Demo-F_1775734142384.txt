Um das in Rust zu automatisieren, müsstest du das Demo-File (.dem) parsen, die Spieler und ihre Teams extrahieren und die Bitmaske berechnen.
Hier ist das Konzept und ein Rust-Code-Beispiel, wie du das umsetzen würdest.
Die Logik für die Automatisierung
Parser nutzen: Du brauchst eine Library, die CS2 (Source 2) Demos lesen kann (z. B. den Wrapper für demoparser2 oder cs2-demo).
Spieler filtern: Du iterierst durch alle Spieler-Entities.
Team Check: Prüfe die Team-ID (m_iTeamNum). In CS2 ist T = 2 und CT = 3.
Slot ID finden: Hole dir den "Entity Index" oder "Player Slot" (0-63).
Bitmaske berechnen: Setze das entsprechende Bit für jeden Spieler des gewünschten Teams.
Rust Code-Beispiel
Dieses Beispiel zeigt die Bitmasken-Berechnung, die der Kern deines Tools wäre. Es nimmt an, dass du bereits eine Liste von Spielern mit ihren Slot-IDs und Team-IDs aus der Demo extrahiert hast.
rust
struct Player {
    name: String,
    team_id: i32, // 2 = T, 3 = CT
    slot: u32,    // Der Slot index (0-63)
}

fn main() {
    // Beispiel-Daten: Ein paar Spieler (Slots sind oft 0-basiert im Parser)
    let players = vec![
        Player { name: "S1mple".into(), team_id: 2, slot: 0 },  // T
        Player { name: "ZywOo".into(),  team_id: 3, slot: 1 },  // CT
        Player { name: "NiKo".into(),   team_id: 2, slot: 2 },  // T
        Player { name: "m0NESY".into(), team_id: 3, slot: 3 },  // CT
        Player { name: "Ropz".into(),   team_id: 2, slot: 60 }, // T (hoher Slot Test)
    ];

    let target_team = 2; // Wir wollen nur Terrorists (Team 2) hören

    let (cmd_low, cmd_high) = calculate_voice_mask(&players, target_team);

    println!("--- Befehle für die Konsole ---");
    println!("tv_listen_voice_indices {}", cmd_low);
    println!("tv_listen_voice_indices_h {}", cmd_high);
}

fn calculate_voice_mask(players: &[Player], target_team: i32) -> (i64, i64) {
    let mut full_mask: u64 = 0;

    for player in players {
        if player.team_id == target_team {
            // Shift 1 um die Slot-Nummer nach links
            // Slot 0 = Bit 0 (Wert 1), Slot 1 = Bit 1 (Wert 2), usw.
            full_mask |= 1 << player.slot;
        }
    }

    // CS2 nutzt zwei Befehle für die 64 Slots.
    // indices: Slots 0-31 (die unteren 32 bits)
    // indices_h: Slots 32-63 (die oberen 32 bits)
    
    // WICHTIG: CS2 interpretiert diese oft als signed 32-bit integers.
    // Wir casten die u64 bits in i64 für die Ausgabe, 
    // aber splitten sie korrekt in zwei 32-bit Blöcke.
    
    let low_part = (full_mask & 0xFFFFFFFF) as i64; 
    let high_part = ((full_mask >> 32) & 0xFFFFFFFF) as i64;

    (low_part, high_part)
}

Worauf du beim Parsen achten musst
Indices: In der Konsole (spec_player) siehst du oft 1-basierte Nummern. Intern (im Code) sind Slots meist 0-basiert. Wenn dein Parser Slot 0 zurückgibt, ist das 1 << 0. Wenn er 1-basiert ist, musst du 1 << (slot - 1) rechnen.
Parser-Empfehlung: Für Rust eignet sich demoparser2, da es extrem performant ist und Bindings bietet, um direkt an CBasePlayerController Daten (wie Team und Name) zu kommen.

Um die Identifizierung von Teams und Spielern in einer CS2-Demo zu automatisieren, musst du auf die sogenannten Entities und deren Properties (Props) zugreifen. In CS2 (Source 2) werden diese Informationen über Controller-Entitäten verwaltet.
1. Die wichtigsten Props für die Automatisierung
Wenn du einen Parser wie demoparser2 (Rust/Python) verwendest, solltest du dich auf die Klasse CCSPlayerController konzentrieren. Diese enthält alle Metadaten eines Spielers.
m_iTeamNum: Der wichtigste Wert für das Team.
2: Terrorists (T)
3: Counter-Terrorists (CT)
m_iszPlayerName: Der Anzeigename des Spielers (hilfreich für Debugging/Logs).
m_steamID: Die eindeutige SteamID64, um Spieler über mehrere Demos hinweg zu tracken.
entity_id / slot: Dies ist der Index (0–63), den du für deine Bitmaske (1 << slot) benötigst. In vielen Parsern wird dies als player_slot oder direkt als Index des Arrays ausgegeben.
2. Umsetzung in Rust (Konzept)
Hier ist ein technischer Ablauf, wie du dies mit demoparser2 in Rust umsetzen kannst:
Schritt 1: Ticks abfragen
Du fragst den Parser nach den Werten für einen bestimmten Zeitpunkt (Tick) in der Demo. Da sich Teams zur Halbzeit ändern können, solltest du die Daten pro Runde oder am Anfang der Demo abrufen.
rust
let wanted_props = vec![
    "m_iTeamNum".to_string(),
    "m_iszPlayerName".to_string(),
    "m_steamID".to_string(),
];

// Parser instanziieren und Daten extrahieren
let df = parser.parse_ticks(wanted_props)?;

Schritt 2: Die Bitmaske generieren
Iteriere durch die Ergebnisse und filtere nach dem gewünschten Team:
rust
fn get_voice_mask_for_team(player_data: &DataFrame, target_team: i32) -> u64 {
    let mut mask: u64 = 0;
    
    // Annahme: player_data enthält Spalten 'slot' und 'm_iTeamNum'
    for row in player_data.iter() {
        let team = row.get("m_iTeamNum");
        let slot = row.get("slot"); // Der interne Index 0-63
        
        if team == target_team {
            mask |= 1 << slot;
        }
    }
    mask
}

3. Warum es keinen "einfacheren" Weg gibt
In CS2 sind Demos technisch gesehen "Multi-User-Replays". Der Server schickt alle Voice-Daten in denselben Stream, und das Spiel entscheidet erst beim Abspielen, welche "Audio-Indices" für deine Ohren geöffnet werden.
Problem: Das Spiel hat keinen eingebauten Automatismus, der erkennt, wer zu welchem Team gehört, um Audio dynamisch zu filtern.
Lösung: Die tv_listen_voice_indices Befehle sind die einzige API, die Valve für diesen Filterprozess bereitstellt.
Pro-Tipp für die Automatisierung
Wenn du die Demo-Dateien in einem Ordner hast, kannst du ein kleines Rust-CLI-Tool schreiben, das:
Den Pfad zur .dem Datei liest.
Den demoparser2 nutzt, um in unter 1 Sekunde das Lineup zu extrahieren.
Dir direkt die fertigen tv_listen_voice_indices Befehle in die Zwischenablage kopiert oder in eine autoexec_demo.cfg schreibt.
Möchtest du Unterstützung beim Setup der Cargo.toml oder einem spezifischen Parser-Crate für dieses Projekt?
Einen guten Einstieg bietet die Dokumentation von demoparser2 auf GitHub, das speziell für High-Speed-Parsing in Rust optimiert ist.
