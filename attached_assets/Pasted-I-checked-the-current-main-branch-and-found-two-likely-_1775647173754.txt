I checked the current main branch and found two likely root causes.

1. Native "In CS2 öffnen" still does not primarily launch through Steam.
Right now in Tauri mode the app calls tauriLaunchCS2(...) and the Rust command launch_cs2 directly spawns cs2.exe with +playdemo.
I want Steam-based launch to be the PRIMARY path in the native app, not only direct cs2.exe spawn.

2. Replay-folder persistence is still fragile.
SettingsPage auto-detect currently only updates the local form state with setForm(...), but does not persist settings immediately.
That means imports/downloads can still run with an empty or outdated demoDirectory unless the user manually clicks Save.

Please fix both properly:

A. Replay folder persistence
- After successful auto-detect in SettingsPage, immediately persist steamPath, cs2Path, and demoDirectory using updateSettings(...)
- Do not require a separate Save click just to make imports/downloads work
- After persisting, refresh the library automatically

B. Import/upload/drop behavior
- Before importing any .dem/.dem.gz/.dem.zst file, validate that settings.demoDirectory is set
- If it is empty, show a blocking German error and stop
- Do not silently import anywhere else
- After import, verify that the returned DemoEntry.directory matches the saved replay folder
- Show the final saved path in the success message

C. FACEIT download behavior
- Before download, validate that settings.demoDirectory is set
- Save all FACEIT downloads/extractions directly into that replay folder
- After download, refresh the local library automatically and ensure the downloaded demo is visible in the library

D. Native CS2 launch
- In Tauri mode, implement Steam launch as the primary path
- Use a real Windows Steam URI / shell open approach first
- Keep direct cs2.exe spawn only as fallback
- Keep clipboard fallback as last fallback
- Explain clearly which method is primary and which is fallback

E. Add explicit debug logging
Please log and surface these exact values when importing/downloading/launching:
- current settings.demoDirectory
- actual dest_dir sent to Rust
- returned DemoEntry.filepath and DemoEntry.directory
- final playdemo argument used
- whether Steam launch or direct cs2.exe launch was attempted
